### מעבר מלא ל‑HTTPS (SSL) בפרודקשן — כל ענפי הפרויקט

**כלל:** בסטייג'ינג מותר עבודה ב־HTTP לפי מדיניות uPress sandbox; **בפרודקשן** חובה **גלישה ציבורית מלאה ב־HTTPS** עם תעודה תקינה, בלי תלות ב־«המשך בכל זאת» בדפדפן. המעבר הוא **חלק בלתי נפרד מפריסת הפרודקשן ומ־M7** (ואימות ב־M5 כהכנה).

| ענף / נכס | מה נסגר בפרודקשן |
|-----------|------------------|
| **אתר WordPress** | כל עמודים, ניהול (`wp-admin`), נכסים סטטיים — רק `https://`; אין mixed content; הפניות `http→https` לפי מארח / תוסף / שרת. |
| **Client Hub (סטטי)** | פריסת `hub/dist/` מקושרת ונבדקת ב־**`https://`** על הדומיין החי (או תת־נתיב מאושר); עדכון קישורים פנימיים אם בסיס השתנה; תיעוד סביבה — `UPRESS_PUBLIC_BASE` וכו' **ב־https** לפרודקשן (ראו [`CLIENT_HUB_APPENDIX_EYAL.md`](../CLIENT_HUB_APPENDIX_EYAL.md)). |
| **REST / אוטומציה** | `UPRESS_WP_REST_BASE` וקריאות API — **רק HTTPS**; Application Password וסודות — לא מעל HTTP פתוח בפרודקשן ([`UPRESS_WORDPRESS_STANDARD_v2.md`](UPRESS_WORDPRESS_STANDARD_v2.md) §7). |
| **FTP / פריסה** | **FTPS** (TLS) כברירת מחדל לפרודקשן לפי §2.1 ב־v2; לא להשאיר מנהג plain FTP על פרודקשן בלי waiver מתועד. |
| **תיעוד ותקשורת** | עדכון קישורי בסיס במאגר (למשל `PROJECT-ENTRY`, `INDEX`, מיילים ללקוח) כך שאינם מפנים ל־`http://` על הדומיין החי. |
| **בדיקות** | צ'קליסט מפורט: [`LAUNCH-CHECKLIST-2026.md`](LAUNCH-CHECKLIST-2026.md) — סעיף «מעבר מלא ל‑HTTPS»; נוהל סטייג'ינג מול פרודקשן: [`../_communication/team_20/STAGING-TLS-VS-PRODUCTION-WORKFLOW-2026-04-02.md`](../_communication/team_20/STAGING-TLS-VS-PRODUCTION-WORKFLOW-2026-04-02.md). |
