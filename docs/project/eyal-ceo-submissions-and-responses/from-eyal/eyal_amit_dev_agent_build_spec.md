## 1. מטרת המסמך

מסמך זה נועד לתת לאייג'נט המפתח מפרט טכני ותבניתי ברור לבניית האתר.

המסמך **לא** נועד לספק קופי סופי או טקסטים סופיים.  
את התוכן המדויק ייוצר בהמשך.

מטרת המסמך היא להגדיר במדויק:

1. אילו עמודים יש באתר
2. מה סדרם בתפריט
3. אילו תבניות נדרשות
4. אילו סוגי רשומה נדרשים
5. אילו שדות/אזורי תוכן יש לכל סוג רשומה
6. אילו פלאגינים/יכולות נדרשים
7. אילו הגדרות גלובליות צריך להכין מראש
8. איך לבנות את המערכת כך שניתן יהיה להזין תוכן בקלות בהמשך

## 2. עקרון פיתוח מרכזי

האתר צריך להיבנות כ־**מערכת תוכן מודולרית**, לא כעמודים קשיחים עם טקסט hard-coded.

כלומר:

- לכל סוג עמוד תהיה תבנית ברורה
- לכל תבנית יהיו אזורי תוכן מוגדרים מראש
- כל אזור תוכן יהיה ניתן לעריכה
- אין לקבע טקסטים סופיים בתוך הקוד
- יש לבנות תשתית גמישה להזנת תוכן עתידית

## 3. עקרונות יסוד לפיתוח

### 3.1 RTL מלא

- כל האתר ב־RTL אמיתי
- כל הרכיבים, טפסים, אקורדיונים, סליידרים, כרטיסים, תפריטים ופופאפים צריכים לתמוך ב־RTL

### 3.2 פונט

- ברירת מחדל זמנית ובטוחה: **Arial**
- אין להשתמש כרגע בפונט עברי "מעוצב" שעלול להישבר
- ניתן להכין תמיכה עתידית להחלפת פונט גלובלית דרך theme settings / customizer / variables

### 3.3 Mobile first

- כל התבניות צריכות לעבוד היטב במובייל
- כל אזור תוכן צריך להיבנות כך שישבר נכון למסך קטן
- תפריט מובייל חייב להיות ברור, קריא ולא עמוס

### 3.4 ביצועים

- להימנע מהעמסה מיותרת של ספריות
- להימנע מאפקטים כבדים
- כל Embed וידאו צריך להיטען בצורה חכמה ככל האפשר

### 3.5 עריכה עתידית

- כל בלוק בדפי הליבה צריך להיות editable
- יש להכין ממשק נוח לעריכת סדר הבלוקים אם אפשר
- אם משתמשים ב־ACF / Gutenberg / builder - לעשות זאת בצורה עקבית ומסודרת

## 13. פלאגינים / יכולות נדרשות

השם המדויק של הפלאגין פחות חשוב.  
מה שחשוב הוא היכולת.  
אם עובדים על וורדפרס, אלו היכולות שצריך:

### 13.1 SEO

צריך כלי שמאפשר:

- title / meta description
- schema בסיסי
- sitemap
- open graph
- noindex בעת הצורך

### 13.2 שדות מותאמים

צריך כלי שמאפשר:

- custom fields
- repeaters
- flexible content
- templates נקיות להזנת תוכן

### 13.3 טפסים

צריך כלי טפסים שמאפשר:

- טופס יצירת קשר
- עמוד תודה
- חיבור ל־email
- חיבור לוואטסאפ / אוטומציות בעתיד

### 13.4 ביצועים

צריך כלי / תשתית עבור:

- caching
- lazy load
- image optimization
- script optimization

### 13.5 נגישות

צריך לוודא שיש תמיכה ב:

- labels
- focus states
- contrast
- keyboard navigation
- עמידה בסיסית בדרישות נגישות

### 13.6 redirects

צריך יכולת להגדיר 301 redirects בקלות

### 13.7 schema custom

צריך אפשרות להזריק / לנהל JSON-LD מותאם אישית

זהו אזור trust מרכזי, ולכן הוא לא יכול להיות hard-coded בלבד.

## 19. Schema - הנחיות טכניות

### 19.1 סוגי Schema שיש להכין

1. Person
2. ProfessionalService או LocalBusiness
3. Service
4. FAQPage
5. Article
6. Course (כשיהיה רלוונטי)

### 19.2 מה לא להשתמש כרגע

- לא `HealthAndBeautyBusiness`
- לא `Circular Breathing` כעוגן מרכזי
- לא `areaServed: Israel` כברירת מחדל
- לא geo מדויק בלי החלטה

### 19.3 sameAs

חובה להכין חיבור דרך `sameAs` לנכסים הרלוונטיים, כולל ויקיפדיה.

## 22. עשה ואל תעשה לצוות הפיתוח

### 22.1 עשה

1. בנה מערכת מודולרית
2. בנה תבניות reusable

4. אל תשתמש בפונטים עבריים בעייתיים
5. אל תבנה תפריט עמוס
6. אל תערבב עולמות תוכן בלי היררכיה
7. אל תסתמך רק על וידאו בלי טקסט
8. אל תשתמש ב־HealthAndBeautyBusiness

10. אל תעשה schema hard-coded לא מדויקת
