## 3. גרסת PHP — התאמה ל־uPress (אין "בלגן" בין מקומי לשרת)

- **חובה:** לבחון בפאנל / תיעוד **uPress** אילו גרסאות PHP זמינות לחשבון, ולבחור **גרסה אחת** לפרודקשן (וסטייג'ינג).
- **חובה:** אותה גרסה (או תמונת Docker מקבילה) **במקומי** — מתועדת ב־runbook צוות 20 וב־`local/` (למשל תג תמונת `wordpress`).
- צוות **20** אחראי לרשום ב־`M2-RUNBOOK-ENV-*.md` את גרסת ה־PHP הסופית ואת מקור האימות (uPress).

## 4. מה ב־Git ומה עולה לשרת — הפרדה קנונית

- **כל הקוד והקבצים שחייבים לחיות במאגר כחלק מהאתר הסופי** — נמצאים תחת עץ **פריסה קנוני** (ראו [`site/README.md`](../../site/README.md)): בעיקר `wp-content` (child theme, `mu-plugins` מותאמים וכו'), לפי מה ש־20 מגדיר ב־runbook.
- **לא עולים לשרת:** `_communication/` (כל צוותים), `docs/` (מסמכי פרויקט), סקריפטי `scripts/` לפיתוח, קבצי `.cursor`, וכל מה שאינו חלק מ־WordPress בפרודקשן.
- **פריסה:** רק תת־העץ המוסכם (או ארטיפקט build) — **לא** העתקת שורש המאגר כולו ל־`public_html`.
