הוסיפו שורות בייצוא ל-Yoast/שרת או בטבלת ניהול. **לא לממש בפרודקשן עד אישור cutover.**

| מקור (URL ישן) | יעד (URL חדש) | סוג | הערה |
|------------------|----------------|-----|------|
| `/shop/` | `/צור-קשר/` או עמוד "רכישה" מידע בלבד | 301 | דוגמה |
| `/shop/cart/` | כנ"ל | 301 | |
| `/shop/checkout/` | כנ"ל | 301 | |
| `/shop/my-account/` | כנ"ל | 301 | |
| עמודי **QR** | — | **ללא 301** | שימור URL למען ספרים מודפסים |
| (המשך למילוי לפי CSV) | | | |

## 4. סיכונים

- **פיצול מותג:** אם נשארים שני מותגים (סטודיו + הוצאה + מופע) באותו דומיין — חשוב מבנה קנוניקל ברור וניווט שלא מבלבל.
- **אובדן טראפיק:** לפני הסרת עמודים עם טראפיק, בדיקה ב-Search Console ויישום 301 לעמוד הכי קרוב.

## 5. קריטריון השלמה

- טבלת 301 מלאה לכל URL שמוסר או משתנה.
