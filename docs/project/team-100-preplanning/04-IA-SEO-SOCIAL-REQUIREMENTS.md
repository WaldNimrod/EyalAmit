## 1. מבנה מידע (IA) — ניווט ראשי מוצע

התאמה ליעד "אתר פשוט": 4–6 פריטים בלבד בתפריט הראשי. **מותגים מרובים באותו דומיין:** הפרדה בחוויית משתמש — סטודיו בראש; הוצאה ומופעים במשנה; פירוט ב-[`IA-WIREFRAMES-AND-EN-LANDING.md`](./IA-WIREFRAMES-AND-EN-LANDING.md).

| פריט | יעד |
|------|-----|
| בית | דף הבית — סיכום שירותים + CTA |
| אודות / אייל | סיפור, ניסיון, קשר למורה (מוקש דהימן) — לפי החלטת תוכן |
| שירותים | תת-תפריט או דף מאגר עם קישורים לעמודי שירות (לימוד, טיפול, סאונד הילינג, בנייה, תיקון, מידע על כלים) |
| בלוג / מאמרים | אופציונלי; רק אם נשארים פוסטים |
| צור קשר | טופס מינימלי + טלפון + מפה (אם רלוונטי) |
| קישור חיצוני | פייסבוק (כבר קיים) — `target="_blank"` + `rel="noopener"` |
| English | עמוד נחיתה EN — `lang="en"`; `hreflang` he/en לפי יישום |

**Wireframe לוגי (טקסט):**

- דף בית: כותרת + פס שירותים (כרטיסים) + אירוע מודגש (אופציונלי) + ציטוט קצר + פוטר עם קשר.
- דף שירות: H1 יחיד, תקציר, פסקאות קצרות, FAQ (אופציונלי), CTA קבוע (דביק במובייל אופציונלי).

### 1.1 המרה ו-UX (השלמה מתכנון מקדים)

- **CTA ראשי אחד** ברור בכל עמוד ליבה; **CTA משני** (למשל השארת פרטים) אופציונלי.
- **Sticky CTA במובייל** לעמודי שירות — מומלץ אם משפר המרה בלי להפר נגישות.
- **מעט עמודים דקים;** עדיפות לעמודי hub חזקים (שירות / השיטה).

## 2. SEO טכני ותוכני

| דרישה | פירוט |
|--------|--------|
| כותרות | H1 אחד לעמוד; היררכיה H2–H3 עקבית |
| מטא | כותרת ותיאור ייחודיים לכל עמוד ליבה; אורך תיאור ~150–160 תווים |
| קנוניקל | תג קנוניקל נכון; ביטול כפילויות אחרי איחוד עמודים |
| Sitemap | XML אחד ברור; שליחה ל-Google Search Console |
| נתונים מובנים | `LocalBusiness` או `HealthAndBeautyBusiness` / `ProfessionalService` (לפי אישור מקצועי), `Person` לאייל, `Article` לפוסטים, `FAQPage` לעמודים עם FAQ, `BreadcrumbList`, `WebSite` עם `SearchAction` רק אם יש חיפוש פנימי |
| שפה וכיוון | `lang="he"` בעמודי עברית; עמוד EN עם `lang="en"`; RTL/LTR נכון |
| קישורים פנימיים | מדף בית לכל שירות ליבה; מבלוג לשירותים רלוונטיים; **מפת כוונות חיפוש** ועמודי cornerstone לפי מחקר מילות מפתח (בכפוף לאייל) |
| Breadcrumbs | מומלץ לעמודי שירות, בלוג, מדור משני — לשפר הקשר ו-SEO |
| FAQ | **לכל עמוד שירות ליבה** — מומלץ (תוכן לא "הבטחה רפואית" רחבה מדי) |
| עמודי מצורף | **noindex** ל-attachment pages ב-WordPress אם אינם נדרשים לאינדקס |
| עמודים לא לאינדקס | תכנון `noindex` לדפי ארכיון דקים / כפילויות — לפי רשימה באישור |

## 5. בדיקות קבלה (לפני השקה)

- Lighthouse SEO ≥ 90 בעמוד הבית (או ללא אזהרות קריטיות).
- שיתוף בדיקה בווטסאפ / פייסבוק — תצוגה נכונה.
- מעבר מקלדט בדף הבית + דף שירות + צור קשר.
