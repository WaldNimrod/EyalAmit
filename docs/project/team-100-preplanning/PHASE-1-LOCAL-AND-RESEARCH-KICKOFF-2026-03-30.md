## 1. תשתית מקומית (WordPress)

**מטרה:** סביבת פיתוח/בדיקה **מקומית** בלבד — בלי לשנות production בלי תיאום.

- **בסיס קוד:** ענף `feature/lean-wp-rebuild-2026` (או ענף משנה ממנו).  
- **אפשרויות טיפוסיות:** Local WP, Docker Compose, MAMP/Local by Flywheel, או העתקת `eyalamit.co.il` ל־`localhost` עם DB מיובא (אם יש dump מאושר).  
- **כללים:**  
  - לא לפרסם שינויי תוכן/מבנה לאתר חי **בלי** סימון בשלב ואישור תפעול.  
  - סיסמאות ו־URLי סטייג'ינג — מחוץ למאגר (ENV מקומי / מנהל סודות).  
- **המשך אחרי בחירת תבנית רזה:** ראו גם [`LEAN-WP-NEXT-STEPS.md`](./LEAN-WP-NEXT-STEPS.md) (יתעדכן מול FINAL, לא מול 05 בלבד).

## 2. סקירות ומיפוי נוסף (מקביל למחקר חשבונית ירוקה)

עבודות אלו **תומכות** באפיון הסופי ואינן חוסמות המשך:

| # | משימה | מקור / כלי | פלט מוצע |
|---|--------|------------|-----------|
| A | **מפת IA מול אתר חי** | השוואה ל־[`SITEMAP-NEW-SITE-v2-DRAFT.md`](./SITEMAP-NEW-SITE-v2-DRAFT.md) | רשימת פערים (עמודים חסרים / כפילויות / שמות תפריט) |
| B | **Inventory / KMD** | [`CONTENT-SSOT-INVENTORY.csv`](./CONTENT-SSOT-INVENTORY.csv) + JSON במאגר `docs/sitemap/` אם קיים | דוח צוות + הכנה לממשק סינון (אם ייבנה) — **אישור אייל אחרי** |
| C | **QR ו-permalink** | [`QR-URL-POLICY.md`](./QR-URL-POLICY.md), [`QR-URL-INVENTORY.csv`](./QR-URL-INVENTORY.csv) | וידוא שאין שינוי slug מתוכנן ללא החלטה |
| D | **הפניות ופלונטרים** | ייצוא redirects מ־SEO plugin / שרת (אם זמין), רשימה ידנית | טבלת "לנקות / לשמר / לדחות" — **בלי** יישום המוני לפני שלב ייעודי |
| E | **קישורים שבורים** | GSC / סריקה פנימית — **כשמופעל** (FINAL דוחה סריקה מלאה "עכשיו" כחובת עצירה, לא כאיסור הכנה) | רשימת URL + קטגוריה (תוכן / באג ישן / חיצוני) |

**הערה:** סעיפים D–E תואמים את סעיף **"רזה"** ב־FINAL (ניקוי הפניות ושבורים) — אפשר לאסוף נתונים מקומית; יישום ב־production לפי תכנון שלב.
