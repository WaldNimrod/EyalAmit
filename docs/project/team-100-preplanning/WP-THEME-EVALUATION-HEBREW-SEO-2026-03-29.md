## 2. שלוש משפחות חלופות

### חלופה A — תבנית בלוקים (FSE) + Child

**תיאור:** תבנית בסיס מ־WordPress.org (למשל משפחת Twenty Twenty-*) או תבנית בלוקים מסחרית קלה, עם **child theme** ל־`theme.json`, טיפוגרפיה, צבעים ותבניות חלקים (headers/footers).

| יתרונות | חסרונות |
|---------|---------|
| HTML נקי, שליטה בפריסה דרך בלוקים; עתיד התאמה ל־WP | עקומת למידה ל־FSE אם לא מכירים |
| תאימות טובה ל־RTL כשהתבנית נבדקת | לא כל תבנית בלוקים מושלמת בעברית — חובה בדיקה |
| פחות תלות ב־CSS מותאם אישית כבד | התאמות עיצוב דורשות היכרות עם `theme.json` |
| ביצועים טובים כשלא משפכים אלמנטים מיותרים | |

**קידום:** כותרות לוגיות, תבניות `template` עקביות, שילוב קל עם Schema מתוסף SEO.

**המלצה פנימית:** **אחת מהחזקות** לפרויקט הזה אם צוות 10 מחזיק ב־FSE.

---

### חלופה B — תמה קלאסית «מהירה» (GeneratePress / Kadence / Astra)

**תיאור:** תבניות ששימשו שנים בתור בסיס ל־אתרים מהירים; שכבת **Customizer** + **Gutenberg**; לרוב יש גרסה חינמית + Pro (לא חובה).

| יתרונות | חסרונות |
|---------|---------|
| מוכחות בשטח, תיעוד רב, קהילה | ממשק כפול: Customizer + עורך בלוקים |
| RTL — לבדוק לכל גרסה; בדרך כלל נתמך | Pro לפעמים מפתה ל«תכונות כבדות» — לשמור על מינימום |
| יציבות עם תוספי SEO | עיצוב «גנרי» בלי child — דורש התאמה |

**קידום:** מבנה מסומן היטב; חשוב להגדיר טיפוגרפיה וריווח ולא להציף במודולים.

**המלצה פנימית:** **ברירת מחדל בטוחה** אם FSE נתפס כסיכון תפעולי לאייל.

---

### חלופה C — תבנית עברית מקומית (ספק ישראלי)

**תיאור:** תבניות המיועדות לשוק המקומי, לעיתים עם תמיכת RTL «מתוך הקופסה» ותרגום ממשק.

| יתרונות | חסרונות |
|---------|---------|
| ניסוח ממשק ולעיתים תמיכה בעברית | איכות קוד וביצועים **משתנים מאוד** בין ספקים |
| התאמה תרבותית (פונטים, כיווניות) | תלות בספק; עדכוני אבטחה — לבדוק היסטוריה |
| | סיכון לתוספים «בונדל» — לבדוק מול 05-PLATFORM §3 |

**המלצה פנימית:** **רק** אחרי סינון טכני: Lighthouse, רשימת תלות, היסטוריית עדכונים, ו**ללא** bundled page builder אסור.
