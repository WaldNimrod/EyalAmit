# נוהל מחייב — חבילת עמוד מלאה עד POC ללקוח

**סטטוס:** מחייב לכל חבילת עמוד (`poc_<pageId>_YYYY-MM-DD` + תוכן מאושר) שמיושמת ב־Hub.  
**עוגן נוהל חבילות:** [`../project/eyal-ceo-submissions-and-responses/from-eyal/canonical_update_pack_2026-04-06/PROCEDURE-MANDATORY-v1.md`](../project/eyal-ceo-submissions-and-responses/from-eyal/canonical_update_pack_2026-04-06/PROCEDURE-MANDATORY-v1.md)  
**דוגמה יישומית:** `hub/src/mockups/poc/poc-kushi-blantis-st-book-kushi-2026-04-06.html` + `st-book-kushi-wp-media/` + `scripts/verify_kushi_intake_poc_parity.py`

## 1. מטרה

לוודא שכל עמוד שמגיע כחבילה מגיע עד ל־**POC שמוכן להצגה ללקוח**: תוכן **מלא ומדויק** למקור המאושר, מדיה **אמיתית** (לא ממלאי מקום מטעים), עימוד **ברור ומכובד** — וכן **הפרדה עקבית** בין תוכן אתר חי לבין הערות/מטא (לקורא אנושי ולסוכן), לפי סעיף 5.

## 2. תוכן טקסטואלי

1. מקור אמת: מסמך מאושר (למשל `YYYY-MM-DD--<pageId>--content--from-eyal.md`) או שדות JSON מקבילים — עם `contentStatus` / `respondent` מתועדים.
2. דף ה־POC ב־HTML משקף **מילה במילה** (או שקילות מפורשת) את הכותרות, הגוף, הערות מהדורה, קטעי ציטוט, וטקסטי CTA — לפי העץ והתבנית.
3. קישורים חיצוניים (רכישה, טפסים): כתובות מדויקות כפי שננעלו ב־`site-tree.json` / החבילה — לא דמו עם URL דמה.

## 3. מדיה

1. **איסור** על הצגת ממלאי מקום (SVG/אפור) ב־POC שמיועד ללקוח כאילו הם התוצר הסופי — אלא אם מסומן במפורש כ־wireframe-only ולא מפורסם כ־«מוכן לאישור».
2. **מקור:** העדיפות היא קבצים מ־**גיבוי האתר הישן** (`eyalamit.co.il-legacy/.../wp-content/uploads/`) או ממפת המדיה ב־`docs/sitemap` — והעתקה לתיקיית מדיה ליד ה־POC (למשל `st-book-kushi-wp-media/cover/`, `.../gallery/`) עם שמות יציבים באנגלית-קישורית.
3. **תיעוד:** `README.md` בתיקיית המדיה — טבלת מיפוי «קובץ ב־POC → נתיב מקור בלגסי».
4. לאחר הוספת בינאריים — להריץ `python3 scripts/build_eyal_client_hub.py` ולוודא ש־`hub/dist` כולל את הקבצים.

## 4. עימוד ותצוגה (חוויית לקוח)

1. **גלריות:** לא לכפות ריבוע או יחס קבוע עם `object-fit: cover` על סט צילומים לא אחיד — זה יוצר עמודות צרות וחיתוכים שמסתירים את התמונה. יש להציג ב־**פרופורציות טבעיות** (`object-fit: contain`, `max-height` סביר, רקע עדין בתא), וברשת **1 עמודה במובייל / 2 עמודות ממסך בינוני** (או היגיון שקול לפי כמות פריטים).
2. **כריכה / hero:** גודל מינימלי שמאפשר הערכה ויזואלית (למשל כריכת ספר ברוחב ~14rem ומעלה בדסקטופ), בלי להקטין מעבר למידה שמקשה על לקוח להבין.
3. **טיפוגרפיה ומרווחים:** עקביים עם פלטת הצבעים והסגנון של האתר; קריאות לפני «צפיפות».

## 5. שכבת מטא מול תוכן עמוד (חובה — אחיד בכל האתר)

כל מה שאינו **תוכן אתר חי** (הערות לצוות, הוראות יישום, מזהי `pageId` / תבנית, נתיבי קבצים במאגר, ניווט בין מוקאפים, מקורות Markdown) חייב להיות **מבודד במכלול ויזואלי וסמנטי אחיד**, כך שגם **אדם** וגם **איגנט** מזהים בו־זמנית מהו תוכן ללקוח ומהו מטא.

### 5.1 חוזה HTML (למימוש ולסריקה אוטומטית)

1. **מכלול עליון:** עוטף יחיד לכל המטא בסוף העמוד (מומלץ), או מכלולים נפרדים — אבל **אותה שפה ויזואלית** בכולם.
2. **חובה על העוטף הראשי:**
   - `data-site-layer="meta"`
   - `data-hub-non-content="true"`
   - מחלקה: `hub-meta-stack` (או שקיל שמתועד בפרויקט).
3. **כותרת גלויה לבן־אדם** בשורה הראשונה של המכלול: למשל «שכבת מטא · לא תוכן עמוד ללקוח» — לא מוסתרת ממסכים (אלא אם קיימת גרסת הדפסה שמסתירה בכוונה).
4. **רמז למכונה:** באותו מכלול, שורה שמציינת במפורש את שמות המאפיינים (`data-site-layer`, `data-hub-non-content`) כדי שסוכנים יוכלו ללמוד את החוזה מהדף.
5. **תת־בלוקים:** כל חלקה פנימית (תגיות, הערות יישום, ניווט פנימי, מקורות) בתוך `hub-meta-block` (או מחלקות `hub-meta-block--tags` / `--nav` וכו׳).

### 5.2 תוכן לעומת מטא בתוך אזור «כמעט אמיתי»

- **כיתובי תמונה ללקוח:** ב־`figcaption` — רק טקסט שיופיע באתר (תיאור קצר בעברית). **נתיבי לגסי / שמות קבצים** — רק ב־`README` של תיקיית המדיה, או בתוך `hub-meta-block`, לא תחת כיתוב הגלריה הציבורי.
- **Placeholder טקסט** (למשל «ישולב כאן קטע מאושר»): חייב להיות מסומן כמטא — `hub-meta-block` עם שורת גבה מסומנת («מטא · לא מאושר») או מיקום בתוך `hub-meta-stack`.
- **חריג:** טקסט שמיועד באמת לציבור (למשל «הרכישה באתר חיצוני») נשאר בגוף העמוד — לא במטא.

## 6. אימות לפני מסירה

1. סקריפט אימות ייעודי לעמוד (כאשר קיים), למשל `verify_*_poc_parity.py` — מחרוזות חובה מהמסמך המאושר + נתיבי מדיה אמיתיים.
2. פתיחה מקומית של `hub/dist/mockups/poc/...` (או מקור תחת `hub/src/mockups/poc/`) ולוודא שכל התמונות נטענות.
3. עדכון אינדקס קליטה / סטטוס (`INGEST-STATUS` או מקביל) כאשר החבילה נסגרה.

## 7. סיכום צ׳ק־ליסט

| שלב | בדיקה |
|-----|--------|
| תוכן | הטקסט ב־POC = המאושר; מטא־דאטה (`pageId`, `slug`, תבנית) נכונה |
| מדיה | קבצים מקומיים מלגסי או ממקור מאושר; README מיפוי |
| תצוגה | בלי crop מטעה; רשת וגבהים סבירים; כריכה בגודל קריא |
| מטא | כל הערות/מזהים/נתיבים פנימיים בתוך `hub-meta-stack` + `data-site-layer="meta"`; כיתובי תמונה ללקוח בלבד |
| בנייה | `build_eyal_client_hub.py`; `dist` מלא |
| אימות | סקריפט parity (אם קיים) עובר |

חריגה מנוהל זה טעונה **תיעוד מפורש** (למשל POC מסומן כ־wireframe-only + סיבה + תאריך).
