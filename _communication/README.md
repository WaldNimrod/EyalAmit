# תקשורת בין-צוותים (`_communication/`)

תיקייה זו מרכזת **דוחות, תוכניות עבודה, סיכומים והודעות צוות** — לא קוד אפליקציה.

## חוקים

1. **כל מסמך שמייצג פלט של צוות** (דוח, תוכנית ספרינט, סיכום בדיקות, מחקר) — רק תחת **`_communication/team_XX/`** של אותו צוות.
2. **קוד** (WordPress, תמה, תוספים) — רק במאגר **[`../eyalamit.co.il-legacy/`](../eyalamit.co.il-legacy/)** לפי מבנה הקבצים שם. אין לשים קוד ב־`_communication/`.
3. **מסמכים קנוניים** (אפיון סופי, SSOT מלא): `docs/project/` ו־`docs/sop/`. צוות 30 מכין טיוטות ב־`team_30/`; שילוב ל־`docs/project` לפי החלטת **צוות 100**.
4. לפני עבודה: [`docs/PROJECT-ENTRY.md`](../docs/PROJECT-ENTRY.md), [`AGENTS.md`](../AGENTS.md), **[`docs/sop/AGENT-WORKSPACE-STANDARD.md`](../docs/sop/AGENT-WORKSPACE-STANDARD.md)** (תקן Cursor/סוכן — מחייב), וקובץ **`onboard_teamXX.md`** של הצוות שלכם (ראו [אונבורד ופרומט ראשון](#onboarding-prompts) למטה).
5. **סוכני AI ב-Cursor:** ברירת מחדל במאגר — **צוות 100** — [`.cursor/rules/eyalamit-team-100-architect-role.mdc`](../.cursor/rules/eyalamit-team-100-architect-role.mdc). **סשן כצוות אחר (10 / 20 / …)** — המשתמש מגדיר במפורש או מדביק בתחילת הצ'אט את **תוכן** קובץ ה־`onboard_teamXX.md` המתאים (טבלה למטה).

## אונבורד ופרומט ראשון לכל צוות

**כללי הפרויקט:** בכל תיקיית `team_XX/` קיים קובץ אחיד **`onboard_teamXX.md`**. הקובץ הוא **הפרומט הקבוע לפתיחת סשן (הקמה והגדרה)** עבור אותו צוות: הוא מגדיר זהות, גבולות, קריאה חובה ומיקום פלטים — **במדויק ובמלואו**, לפני כל משימת ביצוע.

**סדר עבודה מומלץ לסוכן AI**

1. לפתוח את שורש ה-workspace **`EyalAmit.co.il-2026`** (כדי שיטענו `.cursor/rules`).
2. אם **לא** צוין צוות — ברירת המחדל היא **צוות 100**; לקרוא [`team_100/onboard_team100.md`](./team_100/onboard_team100.md).
3. אם נדרש צוות אחר — לקרוא **את קובץ האונבורד של אותו צוות במלואו** (טבלה) או לקבל מהמשתמש הדבקה מלאה שלו בצ'אט.
4. רק אחרי סיום אונבורד — לבקש או לקבל **פרומט משימה נפרד** (מנדט, דוח, ספרינט).

**מפת קבצי אונבורד (קישור ישיר)**

| צוות | תפקיד (תמצית) | קובץ הפרומט הראשון |
|------|----------------|---------------------|
| **100** | אדריכלות, סינתזה, מנדטים | [`team_100/onboard_team100.md`](./team_100/onboard_team100.md) |
| **10** | יישום WordPress, `site/` + legacy | [`team_10/onboard_team10.md`](./team_10/onboard_team10.md) |
| **20** | תשתיות, DB, Git, סביבות | [`team_20/onboard_team20.md`](./team_20/onboard_team20.md) |
| **30** | תיעוד (טיוטות; קנוני באישור 100) | [`team_30/onboard_team30.md`](./team_30/onboard_team30.md) |
| **50** | QA, בדיקות, דוחות איכות | [`team_50/onboard_team50.md`](./team_50/onboard_team50.md) |
| **90** | בקרה על תכניות, מחקר, שערים | [`team_90/onboard_team90.md`](./team_90/onboard_team90.md) |
