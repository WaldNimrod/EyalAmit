# תקשורת בין-צוותים (`_communication/`)

תיקייה זו מרכזת **דוחות, תוכניות עבודה, סיכומים והודעות צוות** — לא קוד אפליקציה.

## חוקים

1. **כל מסמך שמייצג פלט של צוות** (דוח, תוכנית ספרינט, סיכום בדיקות, מחקר) — רק תחת **`_communication/team_XX/`** של אותו צוות.
2. **קוד** (WordPress, תמה, תוספים) — רק במאגר **[`../eyalamit.co.il/`](../eyalamit.co.il/)** לפי מבנה הקבצים שם. אין לשים קוד ב־`_communication/`.
3. **מסמכים קנוניים** (אפיון סופי, SSOT מלא): `docs/project/` ו־`docs/sop/`. צוות 30 מכין טיוטות ב־`team_30/`; שילוב ל־`docs/project` לפי החלטת **צוות 100**.
4. לפני עבודה: [`docs/PROJECT-ENTRY.md`](../docs/PROJECT-ENTRY.md), [`AGENTS.md`](../AGENTS.md), וקובץ **`onboard_teamXX.md`** של הצוות שלכם.

## מפת צוותים

| תיקייה | תפקיד יחיד | פלטים בתיקייה |
|--------|------------|----------------|
| `team_100/` | אדריכלות מערכת, סינתזה, דרישות לצוותים | דוחות ארכיטקטורה, החלטות, מסלולי עבודה |
| `team_10/` | יישום (WP, child theme, `mu-plugins`, `ea_`) | תוכניות יישום, דוחות ספרינט, הערות מיגרציה |
| `team_20/` | תשתיות: DB, DevOps, **Git**, סביבות | runbooks, תוכניות תשתית, הנחיות merge/deploy |
| `team_30/` | תיעוד ודוקומנטציה | טיוטות README, מדריכים, רשימות שינוי |
| `team_50/` | QA, בדיקות, ולידציה | תוכניות בדיקה, דוחות באגים, checklists |
| `team_90/` | מחקר ומיפוי | מחקרים, השוואות, מיפוי סיכונים |

## אונבורד

בכל תיקיית צוות קיים קובץ קבוע **`onboard_teamXX.md`** — יש לקרוא אותו במלואו ואז לבקש את המשימה הראשונה (לא להניח משימה עצמאית).

## מודל מול SSOT הישן

מספרי צוותים ב־`docs/sop/SSOT.md` עשויים עדיין לשקף מבנה ישן (0–4). **מודל העבודה הנוכחי בפרויקט 2026** מוגדר כאן וב־[`docs/ORGANIZATION-TEAMS-2026.md`](../docs/ORGANIZATION-TEAMS-2026.md).
