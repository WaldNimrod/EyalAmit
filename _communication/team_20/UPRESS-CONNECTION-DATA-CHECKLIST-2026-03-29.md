**בעלות:** צוות **20** (runbook).

**רישום סטייג'ינג + PHP + Git (מפורט):** [`UPRESS-STAGING-SITE-RECORD-2026-03-31.md`](./UPRESS-STAGING-SITE-RECORD-2026-03-31.md).

## 1. מה הסוכן / צוות 20 צריך ממך (בלי לפרסם סיסמאות בצ'אט)

מלא ב־runbook פנימי או מסר מוצפן — לא ב-Git ולא ב-Cursor שיתוף ציבורי.


| נושא                       | מה לאסוף                                                                                                                                      |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **PHP**                    | צילום מסך או רשימה מדויקת של **כל הגרסאות** המוצעות במסך "ניהול הגדרות PHP" + **איזו גרסה נבחרה** לסטייג'ינג/פרודקשן (אחרי החלטה).            |
| **סביבות**                 | האם קיימת **התקנת סטייג'ינג** נפרדת (תת־דומיין / חשבון משנה) או שצריך לפתוח אחת; **URL מלא** לכל סביבה שבה נבנה אתר רזה.                      |
| **WordPress חד־לחיצה**     | אם בוצע — **גרסת WP** אחרי ההתקנה, שפת ממשק, והאם זה על סטייג'ינג בלבד.                                                                       |
| **Git (אופציה א׳)**        | URL המאגר (GitHub); **ציבורי או פרטי**; אם פרטי — האם כבר יש PAT למערכת או ייווצר; **איזו תיקייה בשרת** מקושרת ל־Git (נתיב מלא ממנהל הקבצים). |
| **FTP / SFTP (אופציה ב׳)** | Host, פורט, שם משתמש, **נתיב שורש** רלוונטי (למשל עד `wp-content` או `public_html`) — סיסמה במקום סודי בלבד.                                  |
| **מסד נתונים**             | האם יש **phpMyAdmin** בפאנל; קידומת טבלאות (`wp_` או אחר); שם בסיס נתונים (לא סיסמה בטקסט פתוח).                                              |
| **SSL ודומיין**            | תעודה פעילה; האם סטייג'ינג עם `noindex` אפשרי (תוסף / הגדרת ייצור).                                                                           |

## 2. אזהרת ארכיטקטורה — Git מול המאגר שלנו

מאגר **EyalAmit.co.il-2026** כולל `docs/`, `_communication/`, `scripts/` וכו' — **אסור** לשכפל את **שורש המאגר כולו** לתוך `public_html` של WordPress.

**אפשרויות תקינות:**

1. **FTP / SFTP** — להעלות רק את תת־העץ `site/wp-content/` (או מה שיוגדר ב־runbook), או קבצי תמה ספציפיים.
2. **Git ב־uPress** — בדרך כלל מתאים למאגר **קטן וממוקד** שמכיל **רק** את מה שצריך לפרוס (למשל שכפול/מאגר משנה ל־`wp-content` בלבד), **או** תיקיית משנה שנשכפלת לנתיב נכון מתחת ל־`wp-content` — לוודא מול [תיעוד Git של uPress](https://support.upress.co.il/dev/file-manager-git-project/) שהיעד אינו דורס את ליבת WordPress.
3. **שילוב:** Git לתוספים/תמה ממאגר צר + יתר הקבצים ידנית — לפי החלטת 20.

צוות **20** מתעד את המסלול הנבחר ב־`M2-RUNBOOK-ENV-*.md`.

## 3. רשימת בדיקה מומלצת בפאנל uPress (לפני חבילת עבודה מלאה)

סמן במפגש מסך / runbook:

- **הגדרות → ניהול הגדרות PHP** — רשימת גרסאות + בחירה ראשונית (ואז יישור ל־Docker מקומי).
- **התקנת WordPress** (אם רלוונטי) — הושלמה; משתמש מנהל ראשוני מתועד (מחוץ ל-Git).
- **Permalinks** — אחרי כניסה ל־WP: מבנה `/%postname%/` (או שקול) — חובת M2 **20.3a**.
- **גיבויים** — איפה מופעל גיבוי אוטומטי / ידני; תיעוד ל־**20.5a**.
- **מנהל קבצים + Git** (אם נבחר) — ניסוי שכפול לתיקיית בדיקה או אישור תמיכה על מבנה נתיב.
- **FTP** — יצירת חשבון + בדיקת כתיבה לנתון הנכון.
- **Cache / Redis** (אם מופיע בחשבון) — האם פעיל על סטייג'ינג ומה מנקים אחרי deploy.
- **דוא"ל / SMTP** — לא חובה למקומי; בשרת — איזה מסלול uPress מציע (לשלב מאוחר יותר).
