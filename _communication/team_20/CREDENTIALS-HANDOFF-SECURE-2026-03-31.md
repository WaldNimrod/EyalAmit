# העברת פרטי התחברות (FTP / WordPress / DB) — מאובטח

**תאריך:** 2026-03-31 · צוות **20**

מטרה: לתאם עבודה על **סטייג'ינג** (קבצים + לעיתים DB) **בלי** לשים סיסמאות ב־Git, **בלי** להדביק סיסמאות בצ'אט כשאפשר להימנע, ובלי לחשוף את המאגר הציבורי.

---

## 1. FTP / SFTP — מה צריך (ערכים — לא במאגר)

| שדה | למה זה |
|-----|--------|
| פרוטוקל | **SFTP** (מעל SSH) מועדף על **FTP** רגיל אם uPress נותן |
| Host | כתובת שרת מהפאנל |
| פורט | לרוב 22 ל־SFTP, 21 ל־FTP |
| שם משתמש / סיסמה | **רק** במקום סודי (למטה) |
| **נתיב מרוחק** | שורש האתר (למשל עד `public_html`) או נתיב מדויק ש־uPress מציג — חשוב לדעת איפה יושבים `wp-content`, `wp-config.php` |

**היקף קבצים (לפי מה שסיכמתם):**  
עדכונים יכולים לכלול לא רק "תוכן" אלא גם **תבניות, תוספים מותאמים, לעיתים קבצי ליבה** — כל זה עובר ב־**FTP/SFTP** כשמותר בחשבון. **מסד נתונים** עצמו **לא** מתעדכן דרך FTP (חוץ מהעלאת קובץ `.sql` לייבוא ידני). שינויי DB נעשים בדרך כלל דרך **ממשק WordPress**, **ייצוא/ייבוא**, או **phpMyAdmin** בפאנל uPress — לפי נוהל שצוות 20 יגדיר ב־runbook.

## 2. WordPress מותקן — מה צריך לתיאום (בלי סיסמה בצ'אט)

| שדה | מספיק לתעד ב־runbook (לא סודי) | סודי — איפה לשמור |
|-----|--------------------------------|-------------------|
| **URL האתר** | כן — למשל `https://eyalamit-co-il-2026.s887.upress.link` | — |
| **כתובת wp-admin** | כן | — |
| **שם משתמש מנהל** (או משתמש ייעודי לפיתוח) | כן — **רק שם משתמש** | — |
| **סיסמת wp-admin** | לא ב־Git / לא בצ'אט | מנהל סיסמאות או קובץ מקומי (§4) |
| **Application Passwords** (אם בשימוש) | לא לפרסם | אותו עיקרון |

לסוכן AI מספיק לעיתים: "הסיסמה בקובץ המקומי `local/…`" או "בפריט שיתוף במנהל סיסמאות" — **לא** להדביק את הסיסמה בהודעה.

## 3. DB — מה בדרך כלל נדרש

| צורך | איך |
|------|-----|
| גיבוי / שחזור | **phpMyAdmin** או כלי הגיבוי של uPress — פרטי כניסה מהפאנל, לא בהכרח ב־FTP |
| שינויי מבנה / נתונים | דרך **WP Admin**, תוספי מיגרציה, או SQL מבוקר — לא "דחיפת DB ב־FTP" כשגרה |

אם נדרש **ייבוא SQL**: קובץ `.sql` מועלה בזהירות (גיבוי לפני), ו־`local/*.sql` כבר ב־`.gitignore`.

## 4. איך מעבירים סודות **מאובטחים** (מומלץ לפי סדר)

1. **קובץ מקומי בלבד (במחשב שלך, לא ב־Git)**  
   - העתק את [`local/staging.credentials.example.md`](../../local/staging.credentials.example.md) ל־**`local/staging.credentials.md`** (שם הקובץ ב־`.gitignore`).  
   - מלא את הערכים **רק שם**.  
   - ב־Cursor אפשר לומר לסוכן: "הפרטים ב־`local/staging.credentials.md`" — הסוכן קורא מהדיסק המקומי; **אל** תעלה את הקובץ ל-GitHub ואל תשתף אותו במייל לא מאובטח.

2. **מנהל סיסמאות (1Password / Bitwarden וכו')**  
   - פריט מאובטח או **שיתוף חד־פעמי** / **Send** עם קישור שפג תוקף.  
   - לא לשמור את הקישור בתוך המאגר.

3. **הודעה מוצפנת** (Proton Mail, Signal וכו') — חד־פעמי, למקבל ספציפי.

4. **מפגש קצר** — מסירים סיסמה בעל־פה או מדביקים ידנית לקובץ מקומי / vault.

**מה להימנע ממנו:** הדבקת סיסמת FTP או WP **ישירות בצ'אט** — היסטוריית שיחה עלולה להישמר / להיסנכרן; המאגר **ציבורי** — אסור לדחוף לשם שום סוד.
