  the plugin is INERT (no `noindex` injected, page cache not force-disabled).
- **Verified (2026-05-30, staging):** home `X-Robots-Tag: noindex, nofollow` present;

## 4. 301 redirect layer — MECHANISM CHANGE (read before cutover)

- **BLOCKER FINDING (2026-05-30):** Apache `.htaccess` `Redirect`/`RewriteRule` directives
  are **NOT honored** on this uPress stack (nginx front + `AllowOverride None`). Empirically
  every per-page legacy redirect placed in `.htaccess` returned 404; the only "working"
  redirects are PHP `template_redirect` hooks (e.g. `ea_m2_st_canonical_path_redirects()`)
  and WP canonical redirects.
- **Working artifact produced:** `site/wp-content/mu-plugins/ea-w209-legacy-301-redirects.php`
  (generated by `scripts/gen_htaccess_301_from_decisions.py`) — a PHP `template_redirect`
  map of the 25×301 + 2×410 legacy decisions, keyed on the rawurldecode()d path.
- **NOT YET DEPLOYED:** deploying a self-executing mu-plugin to the highest-blast-radius WP
  was withheld pending team_00 approval (mu-plugins are a flagged hazard; the safe
  `.htaccess` deployer cannot cover a PHP mu-plugin). See build report blockers.
- **Cutover action (M7):** after team_00 approves the PHP-mechanism deviation, deploy the
  mu-plugin via `scripts/ftp_deploy_site_wp_content.py` (now registers it) on staging,
  re-run `scripts/final_pre_cutover_check.sh`, then carry to production.
