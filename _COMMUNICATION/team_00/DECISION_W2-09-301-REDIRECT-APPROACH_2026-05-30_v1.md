# DECISION — WP-W2-09 301 application approach

## Topic
How to apply the authoritative 135-decision 301 map for the eyalamit cutover, given the existing
W2-06 `_COMMUNICATION/team_100/tools/htaccess_301_block.txt` carries cruft (conflicting `/shop/ → /books/`
that breaks the live W2-05 catalog; 10 pages lazily 301'd to `/`; risk of catching live Wave2 URLs).

## ⚠ AMENDMENT — B SUPERSEDED BY A (team_00, 2026-05-31; empirical)
**Approach B (.htaccess) is NOT viable on this stack.** team_20 verified empirically across 3 safe-deploy
runs + live probes: the uPress stack (nginx front + Apache `AllowOverride None`) renders ALL per-page
`.htaccess` redirects **inert (404)** — `Redirect`, `RewriteRule`, encoded + decoded all fail. The ONLY
working redirect mechanism is **PHP `template_redirect`** (proven by the live `ea_m2_st_canonical_path_redirects()`
table: `/muzeh/→/muzza/` etc. all 301). **team_00 ratified the switch to Option A (2026-05-31):** apply the
135-map via a generated, guarded PHP mu-plugin `site/wp-content/mu-plugins/ea-w209-legacy-301-redirects.php`
(`template_redirect` priority 0, runs before the canonical table at priority 1), generated by the same
`scripts/gen_htaccess_301_from_decisions.py` from the JSON SSoT. The .htaccess block is retired (left inert).
All the rules below (SSoT-only, de-conflict, no lazy `/`, drop `/shop`, no loops) apply unchanged to the PHP map.
team_100 corrected 2 chain/legacy targets in the generated map → direct canonicals (kushi-blantis→`/books/kushi-blantis/`,
moksha→`/about/moksha/`).

## DECISION (original) — Option B (team_00, 2026-05-30) [SUPERSEDED — see amendment above]
**Clean-REGENERATE `.htaccess` deterministically from the 135-decision JSON as SSoT** (via
`scripts/.../deploy_htaccess_301.py`-style generation), replacing the cruddy hand-edited block.
Dupes/loops/conflicts are eliminated by construction. Keep the clean PHP redirect table
(`ea_m2_st_canonical_path_redirects()`) for site-tree canonical redirects. Server-level .htaccess
matches the prior INGEST plan + team_20 ownership.

### Mandatory rules for the regenerate
1. **Source of truth = the 135-JSON only.** Do not carry forward any rule from the old hand-edited block.
2. **Honor `decided_status` per item:** 27×`301` → redirect; 1×`410` → Gone; 103 `keep`/`qr_protected`
   (incl. all 49 `/qr/qrN/`) → NO rule (stay live 200); 3 `manual` + 1 empty → per §Q2 below.
3. **De-conflict guard:** NO generated rule may match a live Wave2 canonical path (`/`, `/shop`, `/books*`,
   `/treatment`, `/lessons`, `/sound-healing`, `/method`, `/contact`, `/press`, `/en`, `/qr/*`, product slugs,
   `/blog*`). Assert this in the generator (skip + log any collision). Specifically: **drop the legacy
   `/shop/ → /books/` rule** (was the conflict).
4. **No lazy `/` redirects** — empty-target 301s map to their real new-site equivalent (§Q2).
5. Validate htaccess syntax + keep a rollback copy before FTP deploy (atomic; never leave a broken block live).
