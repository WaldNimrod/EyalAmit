## 3. הבנייה — טריאז' מלא + 2 תבניות-תיקון (דרך SSoT)

### 3.1 טריאז' מלא של 400 ה-URL (לא רק 21 המדגם)
לכל 400 ה-URL (9 קבצי ה-txt), הרץ את **אותה שיטה** של המחקר:
1. נרמל path (`trailingslashit`, decode, strip query).
2. בדוק מול 32-הכללים + ה-regex `/Blog/` שבמנגנון החי → in-map? כן/לא.
3. curl **direct** + `-L` מול סטייג'ינג → סטטוס ישיר + סטטוס-מעקב.
4. סווג: `covered` / `false-positive` / `needs-301` / `needs-410` / `needs-content-check` / `out-of-staging-scope`.

פלט: `evidence/s5-03/triage-400/triage.csv` (400 שורות: path / category / in-map? / direct / follow / verdict).

### 3.2 תבנית-תיקון A — תחילית-לגאסי שנייה לא-ממופה
המפה מכסה רק את התחילית `דיגרידו-המרכז-לטיפול-בדיגרידו-סטודי/*`. קיימת **משפחת-URL שנייה עם אותו תוכן**:
`דיגרידו-סטודיו-נשימה-מעגלית-אייל-עמית-2/*` — **0 חוקים**. הוסף ל-SSoT JSON חוקים הממפים כל URL בתחילית השנייה
ל**אותם יעדים** של התחילית המכוסה (במראה).

### 3.3 תבנית-תיקון B — נתיבי `/shop/books/*` חסרים
0 כניסות ל-`/shop/books/*` ב-32-הכללים. הוסף ל-SSoT JSON: `/shop/books/<slug>` → `/books/<slug>` (למשל
`/shop/books/כושי-בלאנטיס/`→`/books/kushi-blantis/`, `/shop/books/צבע-בכחול.../`→`/books/tsva-bekahol/`,
`/shop/books/וכתבת/`→`/books/vekatavta/`), לפי מיפוי ה-slug-ים החי בעץ-האתר.

### 3.4 מסלול-הביצוע (מחייב — SSoT-first, אין hand-edit)
1. הוסף את חוקי §3.2 + §3.3 (+ §4/§5 להלן) כ-**decision entries** ב-`decisions[]` שב-JSON SSoT (135 → 135+N; עדכן `total_items`).
2. הרץ `python3 scripts/gen_htaccess_301_from_decisions.py`.
3. אמת שהוא **regenerated** את שני היעדים, וש-`ea-w209-legacy-301-redirects.php` כעת מכיל את החוקים החדשים (ספירת 301/410 גדלה).
4. **אל תיגע** ידנית ב-mu-plugin או ב-htaccess block — הם build-artifacts. (`collides_canonical()` בסקריפט מגן מהתנגשות מול טבלת ה-canonical.)

**AC-3:** 400/400 URL מסווגים ב-triage.csv; 2 תבניות-התיקון (A+B) מיושמות דרך ה-SSoT + regeneration; מדגם של הפערים
