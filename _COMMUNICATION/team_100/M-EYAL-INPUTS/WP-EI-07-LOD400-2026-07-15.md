### 2.2 סנכרון לקובץ הרינדור (team_10)

**ההחלפה per-item:**
| החלטת אייל | פעולה על `ea-testimonials-fb.json` |
|---|---|
| `approve` | `snippet` ← נשאר (או `snippet_final` אם סופק) |

## 3. יעדי החלפה — GA4 + Clarity

קבצים (mirror): `site/wp-content/themes/ea-eyalamit/inc/analytics-config.json` **+** `hub/data/analytics-config.json` — לעדכן בשניהם.

מצב נוכחי:
```json
"ga4":     { "measurement_id": "G-MRXESK7QJF", "stream_id": "__PENDING_EYAL__" },
"clarity": { "project_id": "__PENDING_EYAL__" },
"status":  "GA4_LIVE_CLARITY_PENDING"
```

| מפתח | שורה (theme) | פעולה |
|---|---|---|
| `ga4.measurement_id` | L3 | **כבר חי** (`G-MRXESK7QJF`, סופק ע"י אייל) — לאשר בלבד |
| `ga4.stream_id` | L4 | `__PENDING_EYAL__` ← Stream ID מ-GA4 (אם אייל מספק) |
| `clarity.project_id` | L9 | `__PENDING_EYAL__` ← Microsoft Clarity Project ID |
| `status` | L13 | ← `LIVE` כאשר Clarity מוזן |

> הבחנה חשובה: **GA4 Measurement ID כבר סופק** וחי; החוסר הפעיל היחיד הוא **Clarity Project ID** (ואופציונלית GA4 `stream_id`). מדריך הקמה קיים: `docs/project/eyal-ceo-submissions-and-responses/to-eyal/2026-06-19--clarity-setup-guide/clarity-setup-guide-he.html`.
