**חשיפה למנועי חיפוש ול־LLM (תמצית אדריכלית):**

- **חיפוש קלאסי:** מבנה IA ברור, כותרות היררכיות, מטא ו־sitemap מ־**Yoast**, **Schema** בסיסי מ־Yoast, ביצועים, פנימיות, תוכן אמין — לפי [`EYAL-TRAFFIC-GROWTH-AEO-GEO-GUIDE-2026-03-29.md`](../../docs/project/team-100-preplanning/EYAL-TRAFFIC-GROWTH-AEO-GEO-GUIDE-2026-03-29.md).
- **AEO / GEO (מנועי תשובה וגנרטיביים):** מנועים ניזונים מטקסט **נקי, מובנה, עקבי**, פחות מ«עמודי בילדר»; **HTML סמנטי** מ־בלוקים/תמה רזה עדיף על שכבות וידג'טים; **לא** להעמיס תוספים שמזהמים DOM או יוצרים כפילות תוכן (AMP כפול, Jetpack בשימוש רחב).
- **מסקנה:** כל תוסף חדש — מבחן: *האם הוא מפשט תוכן מובנה או מוסיף רעש/משקל?*

## 4. מדיניות **מדיה (תמונות)** — חלופות להחלטת נימרוד

**הקשר:** ספרייה גדולה, קבצים ישנים וכבדים, צורך בתהליך **ניתן לתחזוקה מ־WordPress** (אדמין) לאחר המיגרציה.

| חלופה | מה כולל | תחזוקה יום־יומית מ־WP | מתאים למיגרציה המונית | הערות / סיכונים |
|--------|---------|------------------------|-------------------------|------------------|
| **א — ללא תוסף תמונות (מינימום)** | הגדרת **גדלי תמונה** בתמה (GeneratePress + child); הנחיות העלאה לעורכים; **חידוש תמונות מוקטנות** (תוסף uPress) אחרי שינוי גדלים | חלקי — עורך אחראי לגודל קובץ לפני העלאה | **לא** — דורש אצווה **מחוץ ל־WP** (סקריפט, כלי שולחן עבודה, או שירות חיצוני חד־פעמי) לפני/במקביל לייבוא | עלות זמן נמוכה; סיכון שהמיגרציה תשאיר קבצים כבדים אם לא מטפלים מחוץ למערכת |
| **ב — TinyPNG (Compress JPEG & PNG, חבילת uPress)** | דחיסה אוטומטית/אצווה דרך API TinyPNG/TinyJPG | **כן** — מסך אדמין, אצווה, חדשים אחרי הגדרה | **כן** — אידיאלי לספרייה גדולה **אחרי** קבלת מפתח API | **מפתח API מחוץ ל־Git**; תלות בשירות חיצוני; מכסות/עלות לפי חשבון TinyPNG |
| **ג — EzCache (כולל WebP)** | לפי uPress EzCache קשור ל־**Varnish** (ניקוי/ניהול) ולא ל«מנוע קאש מתחרה» בהכרח — ראו **§0** | **כן** אם הופעל EzCache + מודולי התמונה | **כן** — WebP ואופטימיזציה לפי התוסף | לתאם עם **§5**; **לא** לשלב תוסף קאש דפים נוסף; לתאם עם TinyPNG כדי שלא לכפול אופטימיזציה בלי ניקוי/בדיקה |
| **ד — היברידי (מומלץ לעומס מיגרציה)** | **שלב 1:** אופטימיזציה **אצווה מחוץ ל־WP** (ייצוא `uploads`, כלי שורת פקודה, או ספק מיגרציה) → **שלב 2:** **TinyPNG** או הגבלות תמה לשמירה על גודל בעתיד | **כן** בשלב 2 | **כן** בשלב 1+2 | מאזן עלות זמן מול שליטה; שלב 1 חד־פעמי, שלב 2 נשמר בתחזוקה |

**המלצת צוות 100 (טיוטה להחלטה):**

1. **לנעול גדלי תמונה** בתמה/ב־WP לפני «חידוש מוקטנות» — כדי שלא לייצר מיליוני קבצים מיותרים.  
2. **למיגרציה:** חלופה **ד** אם יש אלפי קבצים כבדים; אחרת **ב** עם API ואצווה מהאדמין.  
3. **לא לשלב** TinyPNG מלא + EzCache WebP אגרסיבי **במקביל** בלי בדיקת כפילות וניקוי מטמון — סיכון לכפילות עיבוד ולבלבול בין פורמטים.

**פעולת צוות 10:** אחרי בחירת נימרוד — לעדכן [`M2-IMPLEMENTATION-SUMMARY-2026-04-01.md`](../team_10/M2-IMPLEMENTATION-SUMMARY-2026-04-01.md) §3.2 (מדיניות מדיה) בשורה אחת + קישור לסעיף זה.

## 5. מדיניות **מטמון (קאש)** — מבוסס **תיעוד uPress רשמי** + מדיניות פרויקט

**חובה:** לקרוא תחילה **§0** (קישורים ונתיבי פאנל). להלן סיכום תפעולי — אם יש סתירה, **גובר** המאמר ב־[support.upress.co.il](https://support.upress.co.il/) או הנחיית תמיכה רשמית.

### 5.1 מה מגדיר uPress (לפי מאמרי KB שנבדקו)

| רכיב | היכן בפאנל (לפי המאמרים) | תפקיד |
|------|---------------------------|--------|
| **Varnish Cache** | **ביצועים** → **הגדרות מתקדמות** → **הפעלת Varnish** | מטמון דפים ב־RAM; החרגות (URL / פרמטרים / cookies / UA) ב־**ניהול הגדרות Varnish**. |
| **EzCache** (תוסף WP) | התקנה מ־מאגר התוספים / חבילת uPress | לפי uPress: **ניקוי מטמון** מ־WordPress + **תמיכה בניהול Varnish** — ראו [מאמר Varnish](https://support.upress.co.il/performance/%d7%94%d7%a4%d7%a2%d7%9c%d7%aa-%d7%95%d7%94%d7%92%d7%93%d7%a8%d7%aa-%d7%9e%d7%98%d7%9e%d7%95%d7%9f-varnish-cache/). |
| **ניקוי מטמון (WordPress)** | **פיתוח** → **ניהול כלי אחסון** → **בצע ניקוי מטמון** | לפי [מאמר ניקוי מטמון](https://support.upress.co.il/dev/how-to-clear-cache/). |
| **CDN** | **ביצועים** → **ניהול אזור CDN** | uPress CDN או Cloudflare אישי — [מאמר CDN](https://support.upress.co.il/performance/create-cdn-zone/). |
| **אבחון TTFB** | — | [מאמר TTFB](https://support.upress.co.il/performance/time-to-first-byte/) — תוספים, תמה, DB. |

**שכבות נוספות (Opcode וכו'):** בדרך כלל בשרת — לא מנוהלות מתוך מסמך זה. **Redis / אפליקציה:** אם מופיע בפאנל החשבון — **לתעד ולאמת מול תמיכה**; לא מפורטים במאמרי הביצועים שנסרקו לסעיף זה.

### 5.2 שלב **פיתוח / סטייג'ינג**

| נושא | המלצת 100 |
|------|------------|
| **Varnish + SSL (פרויקט Eyal Amit)** | אם בחשבון **Varnish דורש SSL** והסטייג'ינג אינו במסלול תעודה מלאה — **לא מפעילים Varnish בסטייג'ינג**; **מיועד לפרודקשן** אחרי TLS תקין — ראו [`M2-RUNBOOK-ENV-2026-03-31.md`](../team_20/M2-RUNBOOK-ENV-2026-03-31.md) §13.0. |
| **מצבי מטמון חלופיים בפאנל (סטטי / אפליקצייתי / מותאם)** | לפי דיווח מחזיק פאנל; **לשלב פיתוח:** **מינימום** — ראו הסבר ב־runbook §13.0; לקשר למאמר uPress כשנמצא. |
| **אחרי שינוי תבנית / תוכן / Yoast** | אם התוצאה «נתקעת» — **לנקות מטמון** לפי [מאמר הניקוי](https://support.upress.co.il/dev/how-to-clear-cache/) ו/או כפתור ב־**EzCache** אם מותקן (לפי מאמר Varnish). |
| **Varnish על סטייג'ינג** | רק אם SSL/מדיניות מאפשרים; אחרת **כבוי** — לתעד ב־runbook. |
| **התקנת EzCache** | **אחרי עליה לאוויר** לפי מדיניות הפרויקט (2026-03-29); בפיתוח — לא חובה. |
| **תוסף קאש נוסף (W3TC, Super Cache…)** | **לא** בלי waiver 100 — סיכון כפילות ותוצאות בלתי צפויות מול Varnish. |

### 5.3 שלב **פרודקשן (סופי)**

1. **להפעיל ולכוון Varnish** לפי צורך האתר (החרגות `wp-admin`, עגלות וכו' — ברירות מחדל במאמר).  
2. **EzCache** — לשקול להפעלה אם נדרש **ניקוי שגרתי מ־WP** ויכולות נוספות (WebP וכו') **בהתאם לתיאור התוסף והתמיכה**; לתעד ב־[`M2-IMPLEMENTATION-SUMMARY`](../team_10/M2-IMPLEMENTATION-SUMMARY-2026-04-01.md).  
3. **CDN** — רק אם יש החלטת מוצר/תשתית (עלות, דומיין, TLS); ראו מאמר CDN.  
4. **לא לערבב** מנגנון קאש דפים של צד שלישי עם המחסנית של uPress בלי אישור 100.

**פעולת צוות 20:** ב־runbook — **Varnish: כן/לא** (סטייג'ינג + פרודקשן כשידוע), **EzCache: כן/לא**, צילום או טקסט מ**ניהול כלי אחסון** (ניקוי), וקישור למאמרים ב־§0. ראו [`UPRESS-CONNECTION-DATA-CHECKLIST-2026-03-29.md`](../team_20/UPRESS-CONNECTION-DATA-CHECKLIST-2026-03-29.md) §3.
