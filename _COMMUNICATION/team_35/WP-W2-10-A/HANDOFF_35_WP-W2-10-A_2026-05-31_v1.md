## 4. Key composition decisions
- **Block order:** hero → intro(feeling) → "מה זה"(definition) → "למי מתאים"(fit, pillars) → "איך נראה מפגש"(logistics, bio+real portrait) → "מה ההבדל"(disambiguation, 3-col comparison) → testimonials → faq-mini → disclaimer → closing CTA. Emotional-first, then explanatory, then proof, then ask.
- **Hero treatment:** CSS-gradient **variant_placeholder** — no per-route hero media exists; gradient is the sanctioned no-404 fallback. Single primary CTA.
- **Pillars:** the 5 verbatim "למי זה מתאים" bullets promoted to `.ea-pillars-grid` for scannability (label kicker + verbatim text).
- **Service-comparison:** rendered as **3 columns** (treatment/sound-healing/lessons) in the existing `.ea-service-comparison__grid` hairline pattern; atom CSS ships 2-col, so the 3-col `grid-template-columns` is set in the mockup's scoped style — same atom, same visual language, **no new atom**. Flagged for team_10 (Q4).
- **FAQ-filter:** /treatment has no native FAQ; composed a 3-item mini-FAQ from verbatim on-page sentences, native `<details>` (keyboard, no JS). Sibling routes /sound-healing & /lessons have **real 8-item FAQs** → full list.
- **Testimonials:** 3 real /treatment quotes, sand-circle avatar placeholders (no photos exist), ghost-pill to more.
- **CTA-pill:** repeated hero label in a centered closing band → conversion anchor.
- **Spacing:** `--ea-section-padding` (120px) desktop, `--ea-space-10` (80px) mobile, single breath-divider for cadence.
